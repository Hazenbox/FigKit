# Components

FigKit provides a comprehensive set of React components built with design tokens.

## Available Components

### [Button](/components/button)

The Button component supports multiple variants and sizes.

```tsx
import { Button } from '@figkit/ui';

<Button variant="primary" size="default">
  Click me
</Button>
```

### [Badge](/components/badge)

Use badges to display status, labels, or categories.

```tsx
import { Badge } from '@figkit/ui';

<Badge variant="brand">New</Badge>
```

### [Avatar](/components/avatar)

User avatars with initials, photos, and overflow states.

```tsx
import { Avatar } from '@figkit/ui';

<Avatar size="large" variant="blue" initials="JD" />
```

### [Checkbox](/components/checkbox)

Form controls with various states.

```tsx
import { Checkbox } from '@figkit/ui';

<Checkbox checked label="Checkbox" />
```

### [Tab / Tabs](/components/tabs)

Navigation components for organizing content.

```tsx
import { Tabs, Tab } from '@figkit/ui';

<Tabs items={tabsItems} defaultSelectedId="1" />
```

### [TextInput](/components/text-input)

Form input fields with validation and helper text.

```tsx
import { TextInput } from '@figkit/ui';

<TextInput placeholder="Enter text" label="Label" />
```

## Component Props

All components follow consistent prop patterns:

- **Variants**: Control the visual style (primary, secondary, etc.)
- **Sizes**: Control the size (default, large, wide)
- **States**: Handle disabled, loading, etc.

## Accessibility

All components are built with accessibility in mind, following WCAG 2.1 guidelines.
