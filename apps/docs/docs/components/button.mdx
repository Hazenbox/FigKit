## Variants

Buttons support multiple visual variants to convey different action types and importance levels.

### Primary

Primary buttons are used for the main action in a flow or interface. They use the brand color to draw attention.

<CodePreview code={`<Button variant="primary">Primary</Button>
<Button variant="primary" disabled>Primary Disabled</Button>`}>
  <Button variant="primary">Primary</Button>
  <Button variant="primary" disabled>Primary Disabled</Button>
</CodePreview>

### Secondary

Secondary buttons are used for supporting actions that are less prominent than primary actions. They have a transparent background with a subtle border.

<CodePreview code={`<Button variant="secondary">Secondary</Button>
<Button variant="secondary" disabled>Secondary Disabled</Button>`}>
  <Button variant="secondary">Secondary</Button>
  <Button variant="secondary" disabled>Secondary Disabled</Button>

## Behavior

- Buttons use the semantic `<button>` HTML element
- They have a `transition: 0.1s ease` for smooth state changes
- Disabled buttons have `opacity: 0.4` and `pointer-events: none`
- Icons automatically inherit the button's text color via `currentColor`
- Focus states use a visible border and box-shadow for accessibility

## Design Tokens

Buttons use the following design tokens:

- **Border Radius**: `var(--radius-md, 5px)`
- **Typography**: `var(--font-size-body-medium, 11px)` with `var(--font-weight-medium, 500)`
- **Height Default**: `var(--space-4, 24px)`
- **Height Large**: `var(--space-5, 32px)`
- **Padding Default**: `0 var(--space-2, 8px)`
- **Padding Large**: `0 calc(var(--space-2) + var(--space-1))` (12px)
- **Icon Container**: `var(--space-4, 24px)` × `var(--space-4, 24px)`
- **Icon SVG**: `var(--space-3, 16px)` × `var(--space-3, 16px)`

## Accessibility

The Button component follows WCAG 2.1 accessibility guidelines:

- Uses semantic `<button>` HTML element
- Supports keyboard navigation (Enter/Space to activate, Tab to navigate)
- Proper ARIA attributes for disabled states
- Visible focus indicator with border and box-shadow
- High contrast colors for text readability
- Clear visual distinction between variants and states
- Icons inherit text color for consistent contrast

## ButtonProps

The Button component accepts the following props:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `ButtonVariant` | `'primary'` | Visual style variant of the button |
| `size` | `ButtonSize` | `'default'` | Size of the button |
| `icon` | `ReactNode` | - | Icon element to display |
| `iconPosition` | `'left' \| 'center'` | `'left'` | Position of the icon relative to text |
| `disabled` | `boolean` | `false` | Disabled state |
| `children` | `ReactNode` | - | Button label text content |
| `className` | `string` | - | Additional CSS classes |
| `onClick` | `MouseEventHandler` | - | Click event handler |
| `...rest` | `ButtonHTMLAttributes` | - | All standard button HTML attributes |

## Customization

You can customize button styles by:

1. **Using CSS Variables**: Override design tokens in your theme
2. **Adding Custom Classes**: Use the `className` prop to add custom styles
3. **Extending Styles**: Create wrapper components with additional styling

### Example: Custom Button Styles

```css
/* Custom button styles */
.my-custom-button {
  border-radius: 8px;
  padding: 8px 20px;
  font-weight: 600;
```

## Component Summary

The Button component is a React UI element from the FigKit design system that renders interactive buttons for triggering actions. It's implemented as a semantic button element with extensive variant support, size options, and optional icon integration.

## Component Signature

```typescript
import { Button } from '@figkit/ui';

interface ButtonProps {
  variant?: 'primary' | 'secondary' | 'figjam' | 'destructive' | 'secondary-destruct' | 'inverse' | 'success' | 'link' | 'link-danger' | 'ghost';
  size?: 'default' | 'large' | 'wide';
  icon?: ReactNode;
  iconPosition?: 'left' | 'center';
  disabled?: boolean;
  children?: ReactNode;
  className?: string;
  onClick?: MouseEventHandler<HTMLButtonElement>;
}

<Button {...ButtonProps}>Label</Button>
```

## Quick Start

```tsx
// Basic button
<Button variant="primary">Click me</Button>

// With icon
<Button variant="primary" icon={<PlusIcon />}>Add Item</Button>

// Large size
<Button variant="primary" size="large">Large Button</Button>

// Disabled state
<Button variant="primary" disabled>Unavailable</Button>

// Full width
<Button variant="primary" size="wide">Full Width</Button>
```

## Variant Guide

- **primary**: Blue (#0d99ff) - Main CTA, primary actions
- **secondary**: Transparent with border - Supporting actions
- **figjam**: Purple (#9747ff) - FigJam-specific actions
- **destructive**: Red (#f24822) - Delete, remove, dangerous actions
- **secondary-destruct**: Transparent with red border - Less prominent destructive
- **inverse**: Dark (#2c2c2c) - For light backgrounds
- **success**: Green (#14ae5c) - Confirmations, positive actions
- **link**: Blue text only - Inline link-style buttons
- **link-danger**: Red text only - Destructive link-style
- **ghost**: Transparent - Subtle actions, hover reveals

## Size Guide

- **default**: 24px height, 8px horizontal padding - Standard use
- **large**: 32px height, 12px horizontal padding - Prominent actions
- **wide**: 100% width, 24px height - Mobile, full-width contexts

## Icon Integration

Icons are passed via the `icon` prop. They automatically inherit the button's text color via `currentColor`. Position with `iconPosition`:

```tsx
import { PlusIcon, CheckIcon, TrashIcon } from '@figkit/ui';

// Icon on left (default)
<Button icon={<PlusIcon />}>Add</Button>

// Icon centered
<Button icon={<PlusIcon />} iconPosition="center">Add</Button>
```

## Styling Architecture

Buttons use CSS variables for theming:
- Background: `var(--color-bg-brand)` or variant-specific
- Text: `var(--color-text-onbrand)` or variant-specific  
- Border: `var(--color-border-translucent)` for secondary variants
- Border radius: `var(--radius-md, 5px)`
- Spacing: `var(--space-2)`, `var(--space-4)`, `var(--space-5)`

## Accessibility

- Semantic `<button>` element
- Keyboard support: Enter/Space to activate, Tab to navigate
- Visible focus indicator with box-shadow
- Disabled state prevents interaction (`pointer-events: none`)
- High contrast colors
- Screen reader friendly

## Common Patterns

**Primary Action:**
```tsx
<Button variant="primary" icon={<CheckIcon />}>Save Changes</Button>
```

**Cancel/Secondary:**
```tsx
<Button variant="secondary">Cancel</Button>
```

**Destructive Action:**
```tsx
<Button variant="destructive" icon={<TrashIcon />}>Delete</Button>
```

**Form Submit:**
```tsx
<Button variant="primary" type="submit" size="wide">Submit</Button>
```

**Icon-only (requires aria-label):**
```tsx
<Button variant="ghost" icon={<CloseIcon />} aria-label="Close" />
```

## Implementation Notes

- Uses `<button>` element (not `<a>` or `<div>`)
- Disabled state: `opacity: 0.4`, `cursor: not-allowed`
- Transitions: `0.1s ease` for smooth state changes
- Icons: 24x24 container, 16x16 actual icon size
- Text doesn't wrap (`white-space: nowrap`)
