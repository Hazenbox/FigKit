## Behavior

- Badges are non-interactive by default (`cursor: default`)
- They use `white-space: nowrap` to prevent text wrapping
- Badges have a `transition: all 0.2s ease` for smooth state changes
- The `strong` prop adds a filled background for increased visual prominence
- Icons are only displayed when both `icon` and `iconLead` props are provided

## Design Tokens

Badges use the following design tokens:

- **Border Radius**: `var(--radius-md, 5px)`
- **Typography**: `var(--body-small-fontSize, 9px)` with `var(--body-small-fontWeight, 450)`
- **Height**: `16px`
- **Padding**: `0 var(--space-1, 4px)`
- **Gap**: `var(--space-1, 4px)` between icon and text
- **Icon Size**: `12px × 12px`

## Accessibility

The Badge component follows WCAG 2.1 accessibility guidelines:

- Uses semantic HTML (`button` element)
- Supports keyboard navigation when interactive
- Proper ARIA attributes for disabled states
- High contrast colors for text readability
- Clear visual distinction between variants and states

## BadgeProps

The Badge component accepts the following props:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `BadgeVariant` | `'default'` | Badge variant style |
| `icon` | `ReactNode` | - | Icon element to display |
| `iconLead` | `boolean` | `false` | Show icon when provided |
| `strong` | `boolean` | `false` | Apply strong/bold style with filled background |
| `disabled` | `boolean` | `false` | Disabled state |
| `children` | `ReactNode` | - | Badge content |
| `className` | `string` | - | Additional CSS classes |
| `...rest` | `ButtonHTMLAttributes` | - | All standard button HTML attributes |

## Component Summary

The Badge component is a React UI element from the FigKit design system that renders compact status indicators, labels, or category tags. It's implemented as a semantic button element with extensive variant support and optional icon integration.

## Component Signature

```typescript
import { Badge } from '@figkit/ui';

interface BadgeProps {
  variant?: 'default' | 'brand' | 'component' | 'danger' | 'feedback' | 'figjam' | 'invert' | 'selected' | 'success' | 'variable' | 'variable-selected' | 'warn' | 'merged' | 'archived' | 'menu';
  icon?: ReactNode;
  iconLead?: boolean;
  strong?: boolean;
  disabled?: boolean;
  children?: ReactNode;
  className?: string;
}

<Badge {...BadgeProps}>Content</Badge>
```

## Quick Start

```tsx
// Basic badge
<Badge variant="brand">New</Badge>

// With icon
<Badge variant="success" icon={<CheckIcon />} iconLead>Verified</Badge>

// Strong variant (filled background)
<Badge variant="brand" strong>Featured</Badge>

// Disabled state
<Badge variant="danger" disabled>Unavailable</Badge>
```

## Variant Guide

- **brand**: Primary brand color (#007be5) - Use for important status
- **success**: Green (#009951) - Use for positive/completed states
- **danger**: Red (#dc3412) - Use for errors/critical states
- **warn**: Orange (#b86200) - Use for warnings/caution
- **default**: Neutral gray - Use for general information
- **component**: Purple (#8638e5) - Use for component-related tags
- **variable**: Neutral - Use for variable indicators
- **variable-selected**: Blue border - Selected variable state
- **selected**: Blue border - Selected state
- **merged**: Green border - Merged state
- **archived**: Gray - Archived/inactive state
- **menu**: Dark background - Menu context
- **figjam**: Purple - FigJam-specific
- **invert**: Inverted colors
- **feedback**: Brand color - Feedback context


## Icon Integration

Icons must be provided via the `icon` prop AND `iconLead` must be `true` to display. Icons are 12x12px and appear on the left side of text.

```tsx
import { CheckIcon } from '@figkit/ui';
<Badge icon={<CheckIcon />} iconLead>With Icon</Badge>
```

## Strong Variant

The `strong` prop adds a filled background color matching the variant, with white text for better contrast. Use for increased visual prominence.

## Styling Architecture

Badges use CSS variables for theming:
- Border: `var(--color-border-default)` or variant-specific
- Text: `var(--color-text-default)` or variant-specific
- Background: Transparent by default, filled when `strong={true}`
- Border radius: `var(--radius-md, 5px)`
- Spacing: `var(--space-1, 4px)` or `var(--space-2, 8px)`

## Accessibility

- Semantic `<button>` element
- Disabled state prevents interaction
- High contrast colors
- Keyboard navigable when interactive
- Screen reader friendly

## Common Patterns

**Status Indicators:**
```tsx
<Badge variant="success">Active</Badge>

**Labels/Tags:**

**With Counts:**

```

## Implementation Notes

- Non-interactive by default (`cursor: default`)
- Text doesn't wrap (`white-space: nowrap`)
- Smooth transitions (`transition: all 0.2s ease`)
- Icon only renders when both `icon` and `iconLead` are provided
- Strong variant overrides background and text colors

## Related Components

- Avatar: For user indicators
